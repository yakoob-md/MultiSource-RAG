 Model switching failure — root cause
The bug is a state synchronization problem between three layers. In LegalAssistant.tsx, llmProvider is local component state. When a streaming call starts, it reads the state at that moment. But the FastAPI streaming endpoint in query.py receives llm_provider as a field in the request body — and the call to generate_answer_stream() passes it as provider_name.
The failure happens mid-chat because generate_answer_stream in generator.py has a fallback that defaults to Groq when the fine-tuned model times out or errors, but it does this silently. The frontend has no way to know the switch happened. The user sees an answer but it came from the wrong model.
The second issue: in llm_provider.py, generate_kaggle() has a hardcoded 60-second timeout. The Kaggle bridge (your ngrok tunnel) often takes 45-90 seconds for a cold start. The request times out, generate_kaggle raises an exception, and generate_answer_stream catches it generically and falls through to Groq.

The fix is to emit a typed SSE error event instead of yielding the error as a token, and to add a provider confirmation in the meta event:
python# backend/rag/generator.py — fixed generate_answer_stream

def generate_answer_stream(
    question, multi_result, history=None,
    image_context=None, provider_name="groq", is_legal=False
) -> Generator[str, None, None]:
    
    if not multi_result.all_chunks:
        yield "__NO_RESULTS__"
        return

    is_legal = is_legal or (provider_name == "huggingface")
    messages = _build_messages_rich(
        question, multi_result, history,
        image_context=image_context, is_legal=is_legal
    )
    
    if provider_name != "huggingface":
        # Groq — native streaming
        try:
            client = _get_groq_client()
            stream = client.chat.completions.create(
                model=GROQ_MODEL,
                messages=messages,
                stream=True,
                timeout=GROQ_TIMEOUT,
            )
            for chunk_response in stream:
                token = chunk_response.choices[0].delta.content
                if token is not None:
                    yield token
        except Exception as e:
            # Yield a sentinel the frontend can detect
            yield f"\n\n[SYSTEM: Generation failed — {str(e)}]"
        return

    # Fine-tuned path — simulate streaming from synchronous output
    try:
        answer = llm_provider.generate(messages, mode="finetuned")
        if not answer or not answer.strip():
            yield "[SYSTEM: Fine-tuned model returned empty response. Check Kaggle bridge.]"
            return
        words = answer.split(" ")
        import time
        for i, word in enumerate(words):
            yield word + (" " if i < len(words) - 1 else "")
            time.sleep(0.012)  # slightly slower for readability
    except Exception as e:
        yield f"\n\n[SYSTEM: Fine-tuned model error — {str(e)}. Restart the Kaggle bridge.]"
