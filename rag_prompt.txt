CREATE: backend/rag/query_classifier.py

This module classifies incoming queries into one of three intents and extracts
constraints for retrieval. It must NOT use any ML model — classification is
done via Groq LLM with structured JSON output.

IMPORTS: json, re, from groq import Groq, from backend.config import GROQ_API_KEY, GROQ_MODEL

DEFINE DATACLASS: QueryAnalysis
Fields:
    intent          : str      # "single_source" | "comparison" | "synthesis"
    source_types    : list[str]  # ["legal_statute", "legal_judgment", "web", "youtube", "any"]
    topics          : list[str]  # extracted topics like ["murder", "302", "IPC"]
    ipc_sections    : list[str]  # ["302", "304"] if mentioned
    time_filter     : str | None # "2024" if year mentioned
    language_hint   : str        # "en" | "hi" — detected from query language
    requires_compare: bool       # True if "compare", "difference", "vs" in query
    requires_summary: bool       # True if "common", "all documents", "across" in query
    source_names    : list[str]  # ["IPC", "CrPC"] if explicitly named

DEFINE FUNCTION: classify_query(question: str) -> QueryAnalysis
    Build this system prompt for Groq:
    """
    Classify this legal query. Return ONLY valid JSON with these exact keys:
    {
      "intent": "single_source" | "comparison" | "synthesis",
      "source_types": list of "legal_statute"|"legal_judgment"|"web"|"youtube"|"any",
      "topics": list of topic strings,
      "ipc_sections": list of section numbers as strings,
      "time_filter": year string or null,
      "language_hint": "en" or "hi",
      "requires_compare": boolean,
      "requires_summary": boolean,
      "source_names": list of named sources like ["IPC", "CrPC", "Constitution"]
    }

    Intent rules:
    - single_source: factual question about one law/concept
    - comparison: explicitly asks to compare, contrast, or find differences between two things
    - synthesis: asks for common themes, summary across multiple documents, or general overview

    Query: {question}
    """
    Call Groq with max_tokens=300, stream=False.
    Parse response as JSON. If parsing fails, return safe default:
    QueryAnalysis(intent="single_source", source_types=["any"], topics=[question[:50]], ...)

DEFINE FUNCTION: extract_source_filter(analysis: QueryAnalysis, available_source_ids: list[str]) -> list[str] | None
    - If analysis.source_types == ["any"]: return None (search everything)
    - If analysis.source_names mentions "IPC" or "CrPC" or "Constitution": 
      query MySQL for source IDs where title LIKE '%IPC%' etc.
      Return those IDs.
    - Else return None
    Requires get_connection import from backend.database.connection

The function must have a fallback: if Groq call fails for any reason,
return QueryAnalysis(intent="single_source", source_types=["any"], topics=[], 
ipc_sections=[], time_filter=None, language_hint="en", 
requires_compare=False, requires_summary=False, source_names=[])

Do NOT cache results. Every query must be freshly classified.

Validation checklist:

Test query: "Compare what IPC says about murder vs the 2024 SC judgment" → intent must be "comparison", requires_compare must be True
